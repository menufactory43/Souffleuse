<objective>
Fix the ghost "vér ifi" corruption by recording a deterministic word-boundary flag
in typing history, so `joinHistory` stops guessing with the dictionary, plus a
one-shot legacy sanitation pass that drops already-corrupt entries.

Purpose: The cascade recalls history via `strongCorpusMatch` /
`historyExactSubstringMatch`, which rebuild text with `joinHistory(contextBefore,
accepted)`. For mid-word accepts the boundary intent is KNOWN at accept time but
not persisted, so `joinHistory` guesses via the dictionary — and a TRUNCATED
fragment ("vér"+"ifi"→"vérifi" invalid) is wrongly space-separated → "vér ifi".

Output: a nullable `midWordContinuation` flag end-to-end (entry → SQLite →
joinHistory), a hardened fragment gate that never records truncated sub-words, a
load-time sanitation that purges legacy corruption, and tests + a replay proof.
</objective>

<approach_decision>
**Recording signal source (investigated):** there is no pre-existing authoritative
"this was a mid-word completion" boolean threaded from the cascade to the two
recording sites in `SouffleuseAppDelegate` (full-accept ~1215, partial-accept
`recordPartialAcceptanceToHistoryIfAllowed` ~1297). The structural derivation is
the simpler reliable option and is what this plan uses:

    midWordContinuation =
        OutputFilter.isWordChar(contextBefore.last)   // glues onto current word
        && OutputFilter.isWordChar(accepted.first)
        && (accepted.first is not whitespace)          // implied by isWordChar, kept explicit

When contextBefore is empty, or either boundary char is a space/punctuation, the
accept is a next-word accept → `false`. (Note: `contextBefore` here is the value
ACTUALLY stored, i.e. `SecretHeuristic.contextTail(prefix:)`, so derive the flag
from that same trimmed string the entry carries.)

**Dictionary access inside the Store:** add `SouffleuseTyping` to
`SouffleusePersonalization` deps and use `TypoDetector.isValidWord` directly inside
the actor for the hardened fragment gate and Layer-2 sanitation. `TypoDetector` is
`@unchecked Sendable`; instantiate one shared instance on the actor (private
stored property, lazy). This keeps sanitation INSIDE `TypingHistoryStore.swift`
(privacy invariant) without injecting a closure from outside.
</approach_decision>

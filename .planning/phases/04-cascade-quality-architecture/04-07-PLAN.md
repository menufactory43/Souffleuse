<rationale>
Sous-plan 04-07 de la décomposition Option B. Le human-in-the-loop gate est essentiel parce que :

1. **BUILD-ONLY replay equivalence est inadéquat** : les tests dans subagent env ne chargent pas MLX (modèle non téléchargé, pas de GPU access). La seule détection empirique de régression ghost behavior est un test en conditions réelles sur le dev build signé.

2. **Core Value protect** : CLAUDE.md indique explicitement "Le ghost doit *sembler* aussi instantané et pertinent que Cotypist en usage quotidien. Si le ghost arrive vite mais propose du générique ("Coucou !"), c'est un échec — la qualité contextuelle prime sur la vitesse brute." Une régression silencieuse au niveau du container.perform body (KV cache decision, n-gram bias ordering, filter pipeline timing) violerait cette Core Value sans être détectée par les 223+ tests.

3. **Flag-gated rollback** : si le test empirique trouve une régression, le flag reste comme kill-switch. Pas besoin de revert git complexe.

**Protocole AB recommandé (à exécuter par le user, documenté ici pour reproducibilité)** :

| Phase | Setup | Durée | Apps |
|-------|-------|-------|------|
| Baseline | Souffleuse.app launch standard (no env var) | ≥10 min de frappe | Mail + Notes |
| Test new | Souffleuse.app launch avec `SOUFFLEUSE_USE_MODEL_RUNTIME=1` | ≥10 min de frappe | Mail + Notes (mêmes apps, mêmes types de prompts) |
| Verdict | User notes subjective differences in 04-07-EMPIRICAL-VALIDATION.md | — | — |

**Critères PASS** (subjectifs, user-judged) :
- Ghost arrival timing perceptually equivalent (no noticeable TTFT regression)
- Ghost relevance subjectively equivalent (same kind of completions in same kind of contexts)
- No new false-positive ghosts ("Coucou !" syndrome) introduced
- No new dropped suggestions in contexts where legacy worked

**Critères FAIL** :
- TTFT visibly degraded (>200ms perceptual delay vs baseline)
- Ghost quality degraded in ≥2 distinct contexts
- New crashes or hangs introduced
</rationale>

**Décomposition Option B retenue** :
- **04-05 (ce plan)** : extractions pure-function + lifecycle simple. **Faible risque, aucun changement de comportement.** PVM intact.
- **04-06 (next)** : migration `container.perform` body vers `ModelRuntime.generate(...)` **derrière `SOUFFLEUSE_USE_MODEL_RUNTIME` env flag**. PVM garde `predict_legacy()` comme fallback. AB-testable.
- **04-07 (next)** : Empirical ghost validation (non-autonomous, user gate) + retrait du flag + cleanup PVM façade finale.

Ce plan-ci doit être **byte-identique** dans son impact sur le ghost. Aucun caller PVM n'est modifié.
</rationale>

**Step A — Reproduce session 2026-05-25's protocol.** That session typed a known sequence of characters in a known app to fill `~/Library/Logs/Souffleuse.log` with ~600+ predict events, then derived metrics from log-parse.

Concrete repro:
1. Rotate the existing log out of the way:
   ```bash
   mv ~/Library/Logs/Souffleuse.log ~/Library/Logs/Souffleuse.log.pre-phase3
   ```
2. Build and launch the signed app bundle. **CRITICAL (Warning #1 from plan-checker iter 2):** AX/TCC permissions are bound to the signed `make-app.sh` bundle identity. `swift run Souffleuse` produces an unsigned binary with a different code-identity → macOS denies Accessibility reads → the predict path NEVER fires → the bench measures nothing real. Use:
   ```bash
   cd Souffleuse && bash make-app.sh && open build/Souffleuse.app
   ```
   (If the path is different in your tree, locate the produced `.app` via `find Souffleuse -name "*.app" -path "*/build/*" -type d`.)
3. **Reproduce the typing exercise**: at minimum, type 3-5 paragraphs (~500 characters) in a focused app like Notes or TextEdit. The "post-fix session 2026-05-25" baseline was captured this way. Aim for ≥ 300 `prompt_built` events.
4. Quit the app cleanly (Cmd-Q from the menu bar — gives the log writer a chance to flush).
5. The log is now at `~/Library/Logs/Souffleuse.log`.

**Step B — Extract metrics via jq (or awk fallback).**
```bash
LOG=~/Library/Logs/Souffleuse.log

# Counts
PROMPT_BUILT=$(grep -c '"event":"prompt_built"' "$LOG")
KV_EXTEND=$(grep -c '"event":"kv_cache_extend"' "$LOG")
KV_TRIM=$(grep -c '"event":"kv_cache_trim"' "$LOG")
KV_INVAL=$(grep -c '"event":"kv_cache_invalidate"' "$LOG")

echo "prompt_built=$PROMPT_BUILT extend=$KV_EXTEND trim=$KV_TRIM invalidate=$KV_INVAL"

# prompt_build_ms p50 / p95 (if jq is installed)
jq -r 'select(.event=="prompt_build_ms") | .count' "$LOG" | sort -n > /tmp/pbms.txt
COUNT=$(wc -l < /tmp/pbms.txt | tr -d ' ')
P50_IDX=$(( COUNT / 2 + 1 ))
P95_IDX=$(( COUNT * 95 / 100 + 1 ))
P50=$(sed -n "${P50_IDX}p" /tmp/pbms.txt)
P95=$(sed -n "${P95_IDX}p" /tmp/pbms.txt)
echo "prompt_build_ms: count=$COUNT p50=${P50}ms p95=${P95}ms"
```

**Step C — Capture TTFT.** The current logging infra does NOT emit TTFT to the production log (`Souffleuse.log`). It is held in `PredictorViewModel.ttftMillis` in memory and shown in the UI. Two options:

1. **Preferred**: enable `SOUFFLEUSE_PREDICT_LOG=1` for the run — TTFT is written to `/tmp/souffleuse-predict.log` via the `PredictDebug.log` facility (see `PredictorViewModel.swift:27-48`). Parse `ttft=NNNms` from lines tagged `llm_done_stored`:
   ```bash
   grep "ttft=" /tmp/souffleuse-predict.log | sed -E 's/.*ttft=([0-9]+)ms.*/\1/' | sort -n > /tmp/ttft.txt
   ```
   Compute p50 / p95 the same way as prompt_build_ms.

2. **Fallback**: launch via Xcode in attach mode and read `PredictorViewModel.ttftMillis` via the debugger after each keystroke. Tedious but works if `/tmp/souffleuse-predict.log` is empty (env var was forgotten).

**Step D — Stream completion rate (BLOCKER #1 fix, Choice A from plan-checker iter 2).**

Plan 03-02 added a production-log `Log.info(.predictor, "llm_done_stored", count: tokensProduced)` event at the same site as the dev-only `PredictDebug.log("llm_done_stored", ...)` call. The stream completion ratio is now computable from `~/Library/Logs/Souffleuse.log` directly — **no `SOUFFLEUSE_PREDICT_LOG=1` env-var precondition needed**.

```bash
# Both events live in the production log (audit-safe, count-only):
DONE_STORED=$(grep -c '"event":"llm_done_stored"' "$LOG")
CALLED=$(grep -c '"event":"prompt_built"' "$LOG")
if [ "$CALLED" -eq 0 ]; then
  echo "ERROR: no prompt_built events in log — predict path never fired (Warning #1: did you launch the signed bundle?)"
  exit 1
fi
RATIO=$(awk "BEGIN { printf \"%.2f\", 100 * $DONE_STORED / $CALLED }")
echo "stream_completion_rate=${RATIO}%"
```

This metric is now derivable from the SHIPPING audit-safe log. The dev `/tmp/souffleuse-predict.log` is no longer required for verification (it remains useful for human debugging — e.g. inspecting the actual user text that produced each completion — but is not load-bearing for the phase-verify gate).

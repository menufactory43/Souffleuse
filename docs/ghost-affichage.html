Comment Souffleuse choisit le ghost à afficher

Le texte gris fantôme au curseur — d'où il vient, et pourquoi celui-là plutôt qu'un autre.

Bonjour, comment allez-vous ? En noir : ce que tu tapes. En gris : le ghost proposé. Tab pour accepter, Esc pour rejeter.

Pour chaque frappe, Souffleuse a deux moyens de produire une proposition : se souvenir (rappel de ta propre frappe passée, instantané, sans IA) ou générer (le petit modèle local llama.cpp). Une cascade décide lequel gagne, puis une série de garde-fous empêche d'afficher du bruit ou de faire clignoter le ghost.

1La cascade : se souvenir d'abord, générer ensuite

L0
L1

Étage instant — le rappel corpus (zéro IA, < 1 ms)

Souffleuse cherche dans ta propre prose déjà tapée (historique chiffré) une phrase qui commence pareil. Si le contexte qui matche est assez long, elle propose directement la suite mémorisée. C'est le réflexe « à la Cotypist » : Bonjour, co → mment allez-vous ?, rappelé tel quel, sans rien générer. → event ghost_corpus_fastpath

↓  pas de souvenir assez fort ?
L2

Étage génératif — le modèle local (llama.cpp)

Le modèle génère la suite token par token en streaming. Chaque morceau qui arrive doit passer les 4 garde-fous de la section 3 avant d'être affiché. Le ghost peut donc apparaître, s'allonger, ou être remplacé au fil des tokens. → event ghost_apply_llm

Règle d'or : un souvenir fort a une « confiance » si haute (prior 0.92) que le modèle ne peut que le prolonger, jamais l'écraser. On privilégie ce que TU as réellement écrit.

2Le score : une note de pertinence sur 1

Chaque candidat (souvenir ou token IA) reçoit une note value ∈ [0, 1], produit de trois facteurs :

value = sourcePrior×prefixFit×lengthFit
sourcePrior
∈ [0, 1]

Confiance dans la source. Un souvenir vaut plus qu'une génération IA, qui vaut plus qu'une complétion système.

prefixFit
∈ {0, 1}

Le ghost s'enchaîne-t-il proprement avec ce que tu viens de taper ? Soit oui (1), soit non (0) — un raccord cassé est éliminé.

lengthFit
∈ [0, 1]

Courbe en cloche sur la longueur. 2 à 5 mots = idéal (1.0). Un seul mot ou une phrase trop longue sont pénalisés.

3Les 4 garde-fous, à chaque token généré

Quand le modèle streame, chaque morceau (chunk) traverse cette séquence. Il faut passer les 4 pour s'afficher.

0
Garde mi-mot

Si le curseur est au milieu d'un mot incomplet (« pr… », « C'es… »), on bloque l'IA : le petit modèle devine trop souvent le mauvais mot. Sauf si le mot est déjà complet, ou si le token healing reconstruit le mot entier proprement (« fis »+« cal » = « fiscal »).

ghost_midword_llm_block
1
Plancher de pertinence

Le score doit dépasser le plancher 0.25. En dessous, c'est du bruit — rejeté sans rien afficher.

ghost_gate_block
2
Barre de remplacement (anti-clignotement)

S'il y a déjà un ghost affiché, le nouveau candidat doit le battre d'un facteur ×1.15 pour le remplacer. Sinon on garde l'ancien : le ghost reste stable au lieu de sauter à chaque token. C'est l'event le plus fréquent dans tes logs (885×) — c'est sain.

ghost_keep_under_bar
3
Détection « parasite »

Si un ghost est remplacé moins de 0.8 s après son apparition, il a clignoté trop vite pour être lu : on le marque parasite pour le mesurer (qualité d'expérience).

ghost_classified_parasite
✓
Anti-répétition

Un token qui ne fait que redonner le mot déjà tapé (« bonjour » → « bonjour ») est jeté ; on retombe sur le ghost instant. Dans tes logs, c'est ce qui produit les rafales de ghost_dropped_repeat.

→ affiché

5La courbe de longueur (lengthFit)

Pourquoi 3 mots valent mieux qu'un seul ou que dix : la note par nombre de mots.

Nombre de mots123456789+
lengthFit 0.6 1.0 1.0 1.0 1.0 0.85 0.6 0.6 0.3

Sweet spot 2–5 mots (note pleine). Un mot isolé est faible, une phrase de 9+ mots est fortement pénalisée — trop long à lire en ghost.

6Le ghost en mouvement

Une boucle qui rejoue les cas réels : souvenir instantané, streaming du modèle, candidat rejeté sous la barre, parasite (remplacé trop vite), répétition jetée, puis acceptation au Tab.

En résumé : le souvenir propose vite ce que tu as déjà écrit ; le modèle remplit quand il n'y a pas de souvenir ; le score (source × raccord × longueur) classe les candidats ; et 4 garde-fous empêchent d'afficher un mot incohérent ou de faire clignoter le ghost.

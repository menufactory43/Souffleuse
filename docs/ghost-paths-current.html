1 · Le pipeline, en un coup d'œil

1Poll 50 ms → AXSnapshot (ce qui est focus) → debounce ~15 ms après stabilisation du préfixe.
2routeInstant zéro LLM — tente un ghost instantané depuis ton historique & ton lexique. Si trouvé → affiché direct, le LLM ne pourra que l'ÉTENDRE.
3CompletionCache — hit FIFO (cycle espace↔backspace) + KV cache réutilisé entre frappes.
4Génération LLM — uniquement si l'instant et le cache n'ont rien. Deux familles : mid-mot long-ghost et après-espace streaming.
5Plancher dico filet — quand le gradient d'engagement allait rendre du vide mid-mot, on complète le mot courant (orienté par le greedy).
6Rolling refill (living ghost) — quand le souffle est PLEIN, on régénère les mots à droite pendant que tu consommes à gauche.

2 · Arbre de décision (avec tes flags)

① routeInstant — affiché DIRECTEMENT, sans aucune inférence

Si caret MID-MOT (dernière lettre tapée est une lettre) :

L1 · Rappel corpus fort — ta prose passée (scopée par cluster d'app/registre). Match → ghost source=.history, prior 0.92 (quasi-imbattable). Quality-gate rejette un rappel tronqué.

→ Si le mot partiel est déjà un mot complet (≥4 car. + valide) : on laisse la main au next-word / LLM.

L0 · Lexique appris — tes termes distinctifs que le modèle ignore (Binance, noms clients…), gated capitale + fréquence (~90 % précision). Match → source=.learnedWord.

L0 · Complétion dico système OFF — wordCompleterEnabledRuntime en pause (réactivable via SOUFFLEUSE_WORDCOMPLETER=1). Donc ici : rien.

Si APRÈS-ESPACE / ponctuation / vide :

Rappel corpus fort puis L1 · sous-chaîne exacte d'historique → source=.history.

Aucun match instant → on tombe dans le cache, puis le LLM.

② LLM mid-mot — le cœur (long-ghost + gradient d'engagement)

Déclenché quand : fragment de mot INCOMPLET et l'instant L1 n'a rien rappelé. midWordEscalationEnabled ON, midWordLongGhostEnabled ON ⇒ on prend le long-ghost (une passe greedy healed), PAS la cascade K-branches d'origine (elle ne reviendrait qu'avec SOUFFLEUSE_LONGGHOST_OFF=1).

Gradient d'engagement MW_ENGAGEMENT ON — décide la PROFONDEUR du souffle :

a) Dégénéré structurel — le greedy ne prolonge même pas ton mot (« docu » → dérive) → niveau .zero.
→ PLANCHER DICO NOUVEAU orienté par le greedy. Si une complétion existe → mot figé affiché ; sinon → vide.

b) Fast-accept (P1 ≥ escFastP1 & partial ≥ escMinFastLen) → PLEIN direct, sans lancer les branches.

c) Sinon → K=3 branches (greedy stochastiques, mêmes seeds/temp que l'escalade, early-exit à la majorité) → accord [0,1] via midWordBranchDecision :

AccordNiveauRésultat
≥ 0.8 (MW_ENG_PLEIN)PLEINContinuation pleine (~maxWords) + rolling refill autorisé (living ghost)
≥ 0 (MW_ENG_PRUDENT=0)PRUDENT1 mot (le modal), FIGÉ, pas de rolling
< 0 → impossibleZÉROinatteignable car seuil PRUDENT = 0
⚠️ Conséquence de MW_ENG_PRUDENT=0 : tu as neutralisé la sortie .zero par accord faible → tout accord ≥ 0 donne au moins PRUDENT. Le seul vide mid-mot restant vient du dégénéré structurel (cas a) — et c'est exactement ce que le plancher dico couvre.

③ Plancher dico — le filet (détail)

Quand : uniquement sur .zero mid-mot (LLM s'abstient), mot en cours ≥ 3 lettres. Jamais en fin de phrase, jamais par-dessus un souffle LLM.

Source : NSSpellChecker (FR+EN) via WordCompleter.completion(for:preferring:).

Orienté greedy : choisit le candidat dico qui partage le plus long préfixe avec le mot que le LLM penchait à produire → la bonne conjugaison (« nous mange » + greedy « mangeons » → ons, pas l'infinitif). Greedy vide/en vrille → repli sur le 1ᵉʳ candidat (jamais pire).

Inspecteur : raison engage:floor-dico(structdegen | agree=…). Figé (engagement .prudent).

④ LLM après-espace / next-word — streaming

generateLlama + onLLMChunk = Relevance Gate : un chunk doit passer gateFloor 0.25 et battre l'existant d'un facteur replacementBar 1.15 pour s'afficher.

En pratique ce chemin streaming s'affiche très rarement : le long-ghost mid-mot capte la quasi-totalité des ghosts (mesuré ~0/449 côté streaming, ~250 long-ghost).

ghostStreamEnabled OFF → le long-ghost est peint en one-shot, pas token-par-token.

⑤ Rolling refill — le « ghost vivant »

midWordGhostRollingEnabled ON — seulement quand le niveau est PLEIN (rollingAllowed). À mesure que tu tapes dans le ghost, extendGhost() régénère le(s) mot(s) suivant(s) à droite (profondeur cible 4 mots, recharge sous 2). Il rejoue la même persona/contexte que le souffle initial (ctxPrefix + customInstr + field + biais n-gram appris) → ne dérive plus vers le générique. PRUDENT et le plancher dico sont FIGÉS (pas de roulement).

3 · Ce qui part au LLM (prompt) vs ce qui s'affiche direct

Envoyé à llama.cpp (buildLlamaPrompt)

Modèle base/pt (pas instruct) → continuation brute gauche→droite. Le prompt assemblé, dans l'ordre :

  • Contexte : {customInstr} — ta personnalisation (prose FR, jamais une commande)
  • Few-shot — ta prose passée scopée (examplesInjectionEnabled ON), brute, sans label
  • ctxPrefix — contexte app/fenêtre
  • fieldContext — champ / placeholder / aide
  • beforeCursor — derniers 1024 car. du texte avant le caret, espace final retiré (toujours en DERNIER)

system et afterCursor sont IGNORÉS — un base model ne suit pas d'instruction, et le texte APRÈS le curseur n'entre jamais dans le prompt.

Affiché SANS LLM

  • Rappel corpus (L1) — ta prose passée, telle quelle
  • Lexique appris (L0) — tes termes distinctifs
  • Plancher dico — complétion NSSpellChecker du mot courant
  • Cache — résultats récents (FIFO + KV)

Le biais perso n-gram appris (accepts) influence aussi la GÉNÉRATION LLM via le sampler, quand personalizationStrength > 0 — y compris sur le refill roulant.

5 · Résumé en une phrase

À chaque frappe : on tente d'abord un ghost instant depuis ton historique/lexique (zéro LLM) ; sinon le long-ghost LLM mid-mot génère et un gradient d'engagement (accord de 3 branches) décide s'il déroule un living ghost roulant (PLEIN) ou juste 1 mot figé (PRUDENT) ; et là où le LLM s'abstiendrait au milieu d'un mot, un plancher dico orienté greedy garantit malgré tout un mot valide — sauf en fin de phrase. Le chemin streaming après-espace existe mais ne s'affiche presque jamais.

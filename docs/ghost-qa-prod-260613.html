🔴 Le seul vrai bug : du markup HTML rejoué dans le ghost

Pour déclarer mes impôts j'utilise| → <strong>mot</strong>

Mécanisme (tracé) : l'overlay_shown est émis sans chunk_applied préalable → le ghost vient du chemin instant-recall L1 qui rejoue un verbatim du corpus. Tu as dû coller/saisir du HTML un jour ; l'entrée est ressortie telle quelle. Le filtre banMarkup ne protège que la génération LLM, pas ce chemin de rappel direct — d'où la fuite, identique perso ON et OFF.

✅ Fix proposé (≈ 0 ms, ne touche pas la perso)

Passer le ghost du chemin instant-recall par le même OutputFilter (strip markup / chunk filter) que la sortie LLM, avant affichage. Post-filtre pur → aucun impact sur le budget ~200 ms. Optionnel : assainir les entrées contenant du markup à l'écriture dans l'historique.

Latence

Le ghost est apparu de façon réactive à chaque frappe (fenêtre de stabilisation 1,4 s toujours suffisante). Beaucoup de ghosts proviennent du cache / rappel instant (TTFT négligeable) ; la voie LLM reste calibrée sur la baseline ~200 ms inchangée — aucune régression introduite par ce test.

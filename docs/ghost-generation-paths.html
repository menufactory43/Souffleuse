01 Vue d'ensemble — le ghost en 5 segments #

Une frappe traverse cinq segments. Le LLM n'est que l'un des producteurs possibles, et il est souvent court-circuité par une voie directe (cache, corpus, lexique) plus rapide et plus sûre.

Segment 1 · déclenchement + capture

Poll 50 ms → AXSnapshot → debounce 15 ms

SouffleuseAppDelegate.tick() tourne à 20 Hz. À chaque frappe on lit un AXSnapshot (texte, caretIndex, rect, role…), on extrait le before-cursor, puis un debounce de 15 ms appelle predictor.predict(...). Chaque nouveau préfixe annule le Task en vol (generation counter). SouffleuseAppDelegate.swift · AXClient.swift

Segment 2 · enrichissement

ContextEnricher → prose simple (app · clipboard · OCR)

Actor à cache TTL 5 s par bundle. Assemble app/fenêtre (toujours), clipboard (opt-in, blocklisté), OCR écran (opt-in, off par défaut). Sérialisé en prose inline, jamais en [Label:]. SouffleuseContext/ContextEnricher.swift

Segment 3 · décision

PredictorViewModel — 7 voies, priorité L0 → L1 → cache → LLM

Le cœur : décide si le ghost vient d'une voie DIRECT (lexique, corpus, cache) ou du LLM. La première voie qui réussit gagne. PredictorViewModel.swift · SuggestionPolicy.swift

Segment 4 · prompt + génération

buildLlamaPrompt → llama.cpp (streaming token par token)

Si on tombe sur le LLM : construction d'un prompt texte brut (modèle base/PT), décodage streaming, KV-cache réutilisé entre frappes. LlamaPromptBuilder.swift · ModelRuntime.swift · LlamaEngine.swift

Segment 5 · filtres + affichage

ChunkFilter/OutputFilter → troncature → OverlayWindow (ghost gris)

Chaque token est filtré (markup, écho, dégénéré), tronqué (1 phrase / N mots), puis rendu en gris au caret. Tab accepte, Esc rejette. ChunkFilter.swift · OutputFilter.swift · OverlayWindow.swift

02 Capture — ce qu'on lit, before/after-cursor #

Seul type qui franchit la frontière AX : AXSnapshot (immuable, Sendable). Le « before-cursor » (le préfixe) est String(text.prefix(caretIndex)), capé à 2048 chars côté predictor.

Champ AXSnapshotContenuUsage ghost
text + caretIndexTexte complet du champ + position du caret→ before-cursor : le LLM voit les 1024 derniers chars ; le userTail de 2048 sert au cache/anti-répétition

⚠︎ Le texte APRÈS le curseur n'atteint pas le modèle ghost

Le slot afterCursorSlot (« Suite du texte (à ne pas répéter) : « … » ») est bien construit dans PredictorViewModel et passé à buildLlamaPrompt(afterCursor:) — mais le corps de buildLlamaPrompt ne l'utilise jamais (tout comme le prompt system anglais). Sur le modèle ghost par défaut (Gemma base/PT), seul le texte avant le curseur est continué. LlamaPromptBuilder.swift:88‑128

Gardes qui empêchent tout predict

  • App au premier plan (Souffleuse active) · AX non trusted · champ sécurisé (AXSecureTextField → text=nil) · champ de recherche
  • Bundle blocklisté (Terminal, iTerm, 1Password, Keychain, Zed…) · allowlist = disabled pour l'app
  • Préfixe < 3 chars (trimmed) · caret en plein mot (non-whitespace juste après) · texte « dismissed » par Esc
  • Moteur non chargé (canGenerate == llamaReady == false) · pas encore tapé depuis le focus

05 Le prompt EXACT envoyé au LLM #

Le GGUF ghost par défaut est le modèle base / pré-entraîné (Gemma 3 1B, finetune = pt) — le même fichier que Cotypist. Un modèle base n'a jamais vu de chat-template ni de markers FIM : on le nourrit en texte brut qu'il continue. Tous les chemins ghost de ModelRuntime appellent buildLlamaPrompt.

Structure littérale (chaque bloc optionnel, séparateur \n\n)

// si customInstr (personnalisation) non vide :
Contexte : {customInstructions}

// si examples (prose de l'historique, SimilarHistoryRetrieval) :
{examples bruts, sans label}

// si ctxPrefix (contexte enrichi — voir §6) :
{App …, window "…". Clipboard: …. On screen: ….}

// si fieldContext (annotations du champ) :
Champ : {label}.
Placeholder : « {placeholder} ».
Aide : « {help} ».

Suite du texte (à ne pas répéter) : « {afterCursor} ».   ← CONSTRUIT MAIS JAMAIS UTILISÉ
{system : prompt autocomplete anglais}                   ← CONSTRUIT MAIS JAMAIS UTILISÉ

{beforeCursor}  ← 1024 derniers chars du préfixe corrigé · TOUJOURS EN DERNIER · espace/tab final retiré

Cas minimal — « Bonjour je vous »

Bonjour je vous

Texte brut, rien d'autre. Tokenisé avec BOS : [BOS] Bonjour je vous.

Pourquoi retirer l'espace final ?

Un modèle SentencePiece émet le token suivant avec son espace de tête («  arriver »). Un espace déjà présent fait boucler le greedy. « on va y » → bloque ; « on va y » → « arriver. ».

Cas riche (perso + few-shot + champ)

Contexte : Je m'appelle Gabriel et j'aime la programmation.

Bonjour,
J'aimerais discuter de…

Bonsoir,
Je vous écris au sujet de…

App Mail, window "Re: Réunion Q4".

Champ : zone de message.
Placeholder : « Votre réponse… ».

Bonjour

customInstr est injecté en prose française « Contexte : », jamais comme une commande impérative (un modèle base ne suit pas d'ordres). Le prompt système anglais (« You are an inline autocomplete… ») n'existe que pour un éventuel modèle instruct ; sur le base par défaut il est ignoré. LlamaPromptBuilder.swift:29‑42, 95‑129 · ModelRuntime.swift:386/553/726/846

07 Filtres de sortie & troncature #

Chaque token brut (LLM ou direct) passe par ChunkFilter/OutputFilter avant l'affichage. La génération s'arrête sur fin de phrase ou plafond de mots.

Filtres (rejet / nettoyage)

  • strip prefix overlap — le modèle recopie le préfixe, on l'enlève
  • markup / markdown / emoji / digits — bannis (sampler + post-filtre)
  • écho préfixe / phrase récente → reset (clear ghost)
  • dégénéré (« 1. », bullet, consonne seule) → drop, continue
  • écho d'instruction / de contexte (fuite clipboard/OCR) → drop
  • élision pendante (« l' », « peut- ») → retirée

Troncature (longueur)

  • fin de phrase — coupe à . ? ! … → stop
  • plafond de mots — maxWords mots entiers → stop
  • réglage CompletionLength :
    court = 2 mots · moyen = 3 (défaut) · long = 20
  • emit only when changed — pas de repaint si identique

Affichage & acceptation

  • OverlayWindow — NSPanel non-activant, label gris (tertiaryLabel) ancré juste après le caret ; police = celle du host (via AX) sinon estimée depuis la hauteur du caret.
  • CaretResolver — 4 couches (AX rect → métriques calibrées → défauts → OCR async) pour les hosts sans bounds par char (Chromium, contenteditable).
  • Tab accepte (mot partiel ou ligne entière selon prefs) → AXClient.inject() ; Esc rejette. Cancel-on-keystroke droppe les chunks périmés.
  • GhostInspector (dev) — journalise chaque décision : verdict (AFFICHÉ/GATÉ/DROP/STUB), source (L0/L1/cache/LLM), motif, score (prior·prefixFit·lengthFit).

À retenir

Le ghost n'est pas « toujours du LLM ». Dans l'usage courant, beaucoup de suggestions sortent directement du cache, du corpus personnel ou du lexique — sans toucher llama.cpp — et le LLM ne tranche que quand aucune voie directe ne s'applique. Et même quand le LLM génère, il ne reçoit que les 1024 derniers caractères avant le curseur (+ perso/contexte/exemples) en prose brute : ni le texte d'après le curseur, ni un prompt système, ne lui sont transmis sur le modèle base par défaut. (Le userTail de 2048 chars, lui, ne sert qu'au cache et à l'anti-répétition.)
